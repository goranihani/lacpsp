# Public-Safe Policy

## Upload Allowed

These are safe for a public repository:

- empty templates
- time plans
- role cards
- short reusable prompts
- official-source links and source IDs
- verification status codes
- generic mock drills that do not reveal real competition content
- release checklists

## Upload Forbidden

Never upload:

- real preliminary, main, or final problems
- problem summaries that let a reader reconstruct the problem
- answer drafts or final answers
- AI transcripts or long tool outputs from a real round
- AI-usage reports from a real round
- team names, participant names, law school names, emails, phone numbers, participant numbers, submission links, account IDs, passwords, or screenshots
- venue-provided USB contents or organizer-provided reference packets
- GitHub Issues or PR comments that discuss the live problem

## On-Site Default

Even if this repository is public, do not open it during the main or final round unless organizers expressly say that GitHub access is allowed for this purpose. Treat GitHub as a rehearsal and print-prep tool, not an on-site memory device.

## Why

Main and final rounds are restricted compared with online preparation. The safe posture is:

```text
Before round: public-safe templates can be studied, printed, and rehearsed.
During round: use only allowed AI, allowed searches, and venue-approved local files.
After round: do not publish real problems or answers unless the organizers permit it.
```

## Three-Line Test

Before committing any file, ask:

1. Could this reveal a real problem, answer, AI output, team identity, account, or submission route?
2. Could this be treated as a cloud-stored private working note brought into the venue?
3. Could this mislead a teammate into using a prohibited tool on site?

If any answer is yes, do not publish it.
