# Public-Safe Release Checklist

Run before publishing.

## Content

- [ ] no real competition problem
- [ ] no problem summary that reveals a live or past official problem
- [ ] no answer draft or final answer
- [ ] no real AI transcript
- [ ] no filled AI-usage report from a real round
- [ ] no participant number, team name, school, person name, email, phone, account, password, submission link, screenshot, or venue-provided USB content
- [ ] no GitHub Issue/PR workflow that encourages live-round communication

## Legal Research Hygiene

- [ ] official-source links are source candidates, not final legal conclusions
- [ ] foreign-law source map uses HOLD/CONFIRM language
- [ ] international standards are not described as binding law unless verified
- [ ] free/basic GPT limitations are clear

## On-Site Safety

- [ ] README says not to open GitHub on site unless expressly allowed
- [ ] prompts say not to create citations from memory
- [ ] templates are empty
- [ ] no local-only, output, work, tmp, or submission evidence folder is included
